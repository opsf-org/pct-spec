# Claude Code Instructions

## Branching

When creating branches in this repository, follow the conventions in CONTRIBUTING.md:

- `feat/<description>` — New spec content (fields, sections, extensions)
- `fix/<description>` — Corrections, typos, broken links
- `examples/<description>` — New example tokens or scenarios
- `rfc/<topic>` — Substantive proposals that need discussion

All PRs target `develop`. Never push directly to `main`.

## This Repository

This is the **spec only** — no implementation code belongs here. The reference implementation lives in a separate repository.
